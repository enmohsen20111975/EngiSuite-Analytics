Plan: Fix Pipeline Feature — Real Step-by-Step Calculation Workflow
Context
The current pipeline feature is broken in three critical ways:

No real calculations — The frontend wizard runs a fake Function() eval (sums inputs as fallback). No engineering math happens.
No step chaining — Step outputs do not feed into the next step's inputs. Each step is isolated.
Bad database data — Most pipeline steps in workflows.db have generic names (input_value_1, output_value_1), missing formulas, missing units. Garbage in → garbage out.
Goal: Rebuild pipelines as a proper engineering workflow tool:

User enters inputs for Step 1 → backend calculates → outputs displayed
Outputs auto-fill as inputs for Step 2 → repeat until all steps done
Final screen shows technical report with all inputs, formulas, results, standards
Since the DB data is unreliable, we use a local TypeScript data file instead of the database for pipeline definitions.

Files to Create
1. src/data/engineeringPipelines.ts
Local pipeline definitions — no DB needed. Contains 4 complete engineering pipelines:

Pipeline 1: LV Cable Sizing (Electrical)

Step 1 — Load Analysis: inputs kva_load, power_factor, voltage_ll → outputs current_a, active_power_kw, reactive_power_kvar
Step 2 — Cable Selection: inputs current_a, installation_method, ambient_temp, num_cables → outputs cable_size_mm2, cable_ampacity_a, derating_factor
Step 3 — Voltage Drop Check: inputs cable_size_mm2, cable_length_m, current_a, voltage_ll → outputs voltage_drop_v, voltage_drop_pct, compliant (limit 3% for branch, 5% total per IEC 60364)
Step 4 — Short Circuit Rating: inputs fault_level_ka, cable_size_mm2, clearing_time_s → outputs adiabatic_temp_rise, sc_rating_ka, sc_compliant
Pipeline 2: Power Factor Correction (Electrical)

Step 1 — Existing Power Analysis: inputs active_power_kw, measured_pf → outputs kva_existing, kvar_existing
Step 2 — Correction Sizing: inputs active_power_kw, kvar_existing, target_pf → outputs kvar_required, kvar_to_add
Step 3 — Capacitor Bank Selection: inputs kvar_to_add, voltage_ll → outputs capacitor_kvar, standard_size_kvar, capacitor_current_a
Pipeline 3: Beam Design (Civil/Structural)

Step 1 — Load Analysis: inputs dead_load_kn_m, live_load_kn_m, span_m → outputs total_udl_kn_m, design_load_kn_m, max_moment_kn_m, max_shear_kn
Step 2 — Section Selection: inputs max_moment_kn_m, fy_mpa, steel_grade → outputs required_z_cm3, selected_section, section_z_cm3, section_weight_kg_m
Step 3 — Deflection Check: inputs design_load_kn_m, span_m, e_gpa, selected_section_i_cm4 → outputs max_deflection_mm, allowable_deflection_mm, deflection_compliant
Pipeline 4: HVAC Cooling Load (Mechanical)

Step 1 — Sensible Load: inputs floor_area_m2, window_area_m2, occupants, lighting_w_m2, equipment_w_m2 → outputs sensible_load_kw
Step 2 — Latent Load: inputs occupants, fresh_air_m3_h, indoor_rh_pct → outputs latent_load_kw
Step 3 — Total Load & Selection: inputs sensible_load_kw, latent_load_kw, safety_factor → outputs total_cooling_kw, cooling_tons, selected_unit_kw, cop_est
Each step includes:

formula_display: Human-readable formula string for the report
formula_fn: Actual TypeScript calculation function (not eval, real code)
standard_ref: Code reference (IEC 60364, BS EN 1993-1-1, ASHRAE 62.1, etc.)
description: Engineering explanation of what/why
2. src/routes/localPipelines.routes.ts
New route file with endpoints:


GET  /api/local-pipelines              → list all local pipelines (name, domain, steps count, difficulty)
GET  /api/local-pipelines/:id          → full pipeline with all steps definitions
POST /api/local-pipelines/:id/steps/:stepNumber/calculate  → calculate one step
     body: { inputs: { field: value } }
     returns: { outputs: {...}, formula_display, standard_ref, warnings[] }
POST /api/local-pipelines/:id/report  → generate final technical report
     body: { stepResults: [{step, inputs, outputs},...] }
     returns: { html: "...", json: {...} }
The step calculate endpoint uses the formula_fn directly from local data — no eval, no DB lookup, no strings. Pure TypeScript math.
