Plan: Fix All 4 Tools — Diagram Studio, Logic Simulator, PDF Editor, Data Analysis
Context
Four core tools in EngiSuite are partially broken due to: a backend HTTP method mismatch (PATCH vs PUT), a field name mismatch (state vs data) in save/load, hardcoded userId=1 on all routes, missing npm packages (pdfjs-dist, tesseract.js, papaparse), simulated operations in place of real API calls, and incomplete simulation engine logic. This plan fixes all four tools end-to-end.

Phase 0 — Package Installation
Run in frontend-react/:


npm install pdfjs-dist tesseract.js papaparse
No new backend packages needed (xlsx is already installed at root).

Phase 1 — Cross-Cutting Backend Fixes (All 4 tools depend on these)
1.1 — Add PATCH route to canvas.routes.ts
File: src/routes/canvas.routes.ts (lines 96–118)
Change router.put('/:id', handler) to handle both methods:


router.route('/:id').put(handler).patch(handler);
This fixes the 404 error caused by canvasService.js calling api.patch().

1.2 — Fix state vs data field name mismatch
File: src/routes/canvas.routes.ts
In POST handler (line 41) and PUT/PATCH handler (line 99), update destructuring to accept both field names:


const { name, type, data: dataField, state, thumbnail, isPublic } = req.body;
const data = dataField ?? state;
This fixes silent save failures across all 4 tools.

1.3 — Fix hardcoded userId = 1
Files:

src/routes/canvas.routes.ts (lines 18, 40)
src/routes/analytics.routes.ts (lines 19, 44)
src/routes/vda.routes.ts (line 59)
Create src/middleware/auth.middleware.ts with an optionalAuth middleware that reads the Authorization: Bearer <token> header, verifies with jwt.verify(), and sets req.userId = payload.userId. Falls through without error if no token.

Replace const userId = 1; with const userId = req.userId ?? 1; in all above files.

Apply optionalAuth in src/server.ts before the route mounting for canvas, analytics, and vda routes.

1.4 — Fix handleLoad in all 4 page files
Files: DiagramStudioPage.jsx (~line 1120), LogicSimulatorPage.jsx (~line 965), PDFEditorPage.jsx (~line 760), VisualDataAnalysisPage.jsx (~line 825)

The backend GET /api/canvas/:id returns the Prisma row where canvas state is in field data, but all frontend handleLoad functions read data.state. Fix each to:


const canvasState = typeof data.data === 'string' ? JSON.parse(data.data) : data.data;
const state = canvasState?.state ?? canvasState;
// then use `state` instead of `data.state`
